# /file — 로컬 파일 분석 & Obsidian 저장

파일 경로를 받아 내용을 읽고, 구조화된 한국어 노트로 정리하여 Obsidian에 저장한다.
여러 파일 경로를 받으면 병렬로 처리한다.

시작 전에 Read로 아래 두 파일을 읽어 규칙·포맷을 파악한다:
- `${CLAUDE_PLUGIN_ROOT}/references/config.md` — 공통 변수·저장 절차
- `${CLAUDE_PLUGIN_ROOT}/references/note.md` — 통일 노트 포맷

---

## Phase 0: 파일 경로 확인

`$ARGUMENTS`에서 파일 경로를 추출한다.

- 공백 또는 쉼표로 구분된 여러 경로 → **다건**
- 단일 경로 → **단건**

경로가 없으면 AskUserQuestion으로 요청한다:

```
분석할 파일 경로를 입력해주세요.
(여러 파일은 공백으로 구분: /path/a.md /path/b.pdf)
```

---

## Phase 1: 단건/다건 분기

- **단건:** [파일 분석 & 작성] 서브플로우 실행 → Phase 5
- **다건:** 모든 파일에 대해 [파일 분석 & 작성] 서브플로우를 병렬로 실행 → Phase 5

## [파일 분석 & 작성] 서브플로우

### A. 파일 읽기

`Read`로 해당 경로의 파일을 읽는다.

- PDF: `pages` 파라미터를 활용 (전체 페이지 먼저 시도, 10페이지 초과 시 1-10 페이지부터 시작)
- 읽기 실패 시 해당 파일 건너뛰고 오류 기록

### B. 내용 분석

파일 확장자에 따라 적절한 방식으로 분석한다:

| 확장자                        | 분석 방식                                |
| ----------------------------- | ---------------------------------------- |
| `.md`, `.txt`                 | 전체 내용 구조화 요약                    |
| `.pdf`                        | 섹션별 핵심 내용 추출                    |
| `.py`, `.js`, `.ts` 등 코드   | 기능 요약, 주요 함수·클래스 설명         |
| `.json`, `.yaml`, `.toml`     | 설정 항목 목록화, 주요 값 설명           |
| 기타                          | 파일 내용 전체를 구조적으로 정리         |

### C. 노트 작성

`references/note.md`의 통일 포맷으로 한국어 노트를 작성한다.

**`{CONTENT}` 구조 (file 전용):**

```markdown
## 주요 내용

{파일 유형에 맞는 분석 결과 섹션별 정리}

### {섹션 제목}
- {핵심 포인트}
```

**변수 채우기:**
- `{SOURCE}`: 파일 전체 경로
- `{TAGS}`: 내용 주제 태그 + `#file`
- `{SUBJECT}`: 파일 핵심 주제 2-3줄
- `{인사이트}`: 가장 중요한 인사이트 2-5개

작성 규칙: `references/config.md`의 **작성 규칙**을 따른다.

### D. 저장

`references/config.md`의 **저장 절차**를 따른다. 저장 경로: `{VAULT_BASE}/file/`

## Phase 5: 완료 메시지

### 단건 처리 완료

```
✅ 파일 노트가 저장되었습니다
   파일: {원본 파일 경로}
   경로: {저장된 파일의 전체 경로}
   태그: {태그 목록}
```

### 다건 처리 완료

```
✅ 파일 정리가 완료되었습니다
   저장 경로: {VAULT_BASE}/file/
   처리한 파일: {N}개

| 파일 | 저장 경로 | 상태 |
|------|-----------|------|
| {파일명} | {경로} | 저장완료 |
| {파일명} | {경로} | 실패 |
```
