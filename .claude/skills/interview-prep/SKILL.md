# Interview Prep

## 사용자 프로필
- **직군**: Frontend Developer
- **경력**: 7년차

---

## Step 1: 세션 시작

대화 시작 시 순서대로 진행한다.

### 1-1. 지원 정보 수집 (선택)

다음을 물어본다. 없으면 건너뛴다:
- 지원 회사명
- JD(Job Description) 또는 주요 요구사항

### 1-2. 이력서 선택

어떤 이력서를 기준으로 할지 선택 받는다:

**A. 최신 버전 (웹)**
- WebFetch로 `https://heeyounggoo.github.io/resume` 내용을 가져온다

**B. 구버전 (PDF)**

아래 순서로 캐시를 확인한 후 필요할 때만 PDF를 읽는다. PDF를 매번 읽으면 느리고 불필요하기 때문이다.

1. `tmp/resume_cache.md` 존재 여부 확인
2. 캐시가 있으면: Bash로 PDF 수정 시간을 확인한다
   ```bash
   stat -f "%m" references/구희영_경력기술서.pdf
   ```
   캐시 파일 상단의 `pdf-mtime` 값과 비교한다.
   - **일치(변경 없음)**: 캐시 내용을 그대로 사용한다 → Step 2로 진행
   - **불일치(파일 변경됨)**: 3번으로 진행
3. 캐시가 없거나 PDF가 변경된 경우: PDF를 읽고 `tmp/resume_cache.md`에 저장한다

`tmp/resume_cache.md` 형식:
```
pdf-mtime: {stat으로 얻은 수정 시간 unix timestamp}

{PDF에서 추출한 이력서 내용 전문}
```

파일이 없으면 안내: "references/ 폴더에 PDF 파일을 추가해주세요"

이력서를 성공적으로 로드한 후 주요 영역(프로젝트, 기술 스택, 경험 등)을 파악해 두고 Step 2로 진행한다.

---

## Step 2: 예상 질문 & 답변 수집

### 2-1. 질문과 답변이 있는 경우
예상 질문과 답변을 입력받고 Step 3으로 진행한다.

### 2-2. 질문이 없는 경우
1. 이력서에서 주요 영역을 추출해 목록으로 제시한다
   - 예: 프로젝트 경험, 기술 스택, 트러블슈팅, 협업/리더십, 성장 과정 등
2. 사용자가 준비하고 싶은 영역을 선택한다
3. 선택한 영역에서 예상 질문 1가지를 제시한다
4. 그 질문에 대한 답변을 입력받는다

---

## Step 3: 질문 검증 & 답변 분석

### 3-1. 질문 검증

아래 기준으로 이 질문이 면접 대비에 의미 있는지 판단한다:

| 기준 | 설명 |
|------|------|
| **이력서 연관성** | 이력서의 특정 경험/프로젝트/기술과 연결해 답변할 수 있는가? |
| **경력 수준 적합성** | 7년차 FE 개발자에게 적합한 깊이와 수준인가? |
| **차별화 가능성** | 이력서의 강점이나 특이한 경험을 어필할 수 있는 질문인가? |
| **JD 매칭도** | JD가 있을 때, 포지션 요구사항과 연관된 질문인가? |
| **구체적 답변 가능성** | 추상적 답변이 아닌 실제 사례로 답변 가능한가? |

**검증 결과 처리:**
- **의미 없는 질문**: 이유를 간략히 설명하고, 더 나은 질문 1-2개를 제안한다. 원래 질문으로 계속할지 제안 질문으로 바꿀지 사용자에게 확인한다.
- **의미 있는 질문**: 3-2로 진행한다.

### 3-2. 답변 분석 & 개선

답변을 아래 기준으로 검증하고 개선한다:
- 이력서 기반으로 논리적인가?
- 기술적으로 정확한가?
- 질문의 핵심에서 벗어나지 않는가?
- 7년차 수준에 맞는 깊이인가?

**출력 형식 (이 형식을 그대로 사용할 것):**

```
예상 질문: {예상 질문}
기존 답변: {나의 답변}
최종 답변: {수정된 답변}
주요 확인 필요:
    {좋았던 부분} - 있다면 작성
    {부족한 부분} - 있다면 작성
    {수정한 부분} - 있다면 작성
예상 꼬리 질문: {예상 꼬리 질문 2-3개}
```

분석 완료 후 Step 4로 진행한다.

---

## Step 4: 다음 단계 선택

분석 완료 후 사용자에게 다음 중 선택 받는다:

1. **이어서 대화** - 현재 Q&A에 대해 더 논의
2. **꼬리 질문 답변** - 예상 꼬리 질문 중 선택 후 답변 준비
3. **새로운 예상 질문** - 새 질문으로 Step 2부터 재시작

### 4-1. 저장 시점

이 질문에 대한 논의가 완전히 끝난 시점에 `tmp/qa_log.md`에 저장한다. 이유는 이어서 대화하는 동안 답변이 더 다듬어질 수 있기 때문이다.

- **2번 또는 3번 선택 시**: 이전 Q&A가 완료된 것으로 판단하고 저장 후 진행
- **1번 선택 후 이어서 대화 중**: 사용자가 "다른 질문", "다른 내용 연습", "대화 종료", "새로운 질문" 등 이 질문을 끝내겠다는 의사를 표현할 때 저장 후 Step 4 선택지를 다시 제시한다

### 4-2. 꼬리 질문 선택 시
- 생성된 예상 꼬리 질문 목록을 보여준다
- 사용자가 선택한 꼬리 질문에 대한 답변만 입력받는다 (질문 검증 단계 생략)
- Step 3-2 답변 분석으로 진행한다

## Step 5: 세션 메모리

### 5-1. 저장 위치

```
tmp/
├── session.md    # 세션 메타 정보 (세션 시작 시 1회 생성)
└── qa_log.md     # Q&A 기록 (Q&A 완료마다 누적)
```

### 5-2. session.md 형식

```markdown
# 면접 대비 세션

- 날짜: {오늘 날짜}
- 이력서: {최신/구버전}
- 지원 회사: {회사명 또는 미입력}
- JD 키워드: {주요 키워드 또는 미입력}
```

### 5-3. qa_log.md 누적 형식

Q&A 완료마다 아래 블록을 추가한다:

```markdown
## Q{번호}: {질문 요약}

- 원본 답변: {기존 답변 요약}
- 최종 답변: {개선된 답변}
- 꼬리 질문: {생성된 꼬리 질문들}

---
```

### 5-4. 세션 종료

사용자가 종료를 요청하면:
1. `tmp/` 내의 `session.md`, `qa_log.md` 삭제 여부를 확인한다
2. 동의 시 해당 파일만 삭제한다
3. `resume_cache.md`는 삭제하지 않는다 - 다음 세션에서 재사용하기 위해 유지한다

---

## 주의사항

- 이력서에 있는 실제 경험만 기반으로 답변을 보강한다. 없는 경험을 추가하지 않는다.
- 개선된 답변은 사용자의 말투와 스타일을 최대한 유지한다.
- 기술적 검증 시 확신이 없는 내용은 "확인 필요"로 표시한다.
- 답변이 지나치게 길어지지 않도록 실제 면접 환경(1-3분 답변)을 기준으로 한다.
