## Step 1: 세션 시작

대화 시작 시 순서대로 진행한다.

### 1-1. 이력서 선택

AskUserQuestion 도구로 선택지를 제시한다:

```
질문: "어떤 이력서로 준비할까요?"
옵션:
  - A. 최신 버전 (웹): 웹에서 최신 이력서를 가져옵니다
  - B. 구버전 (PDF): 로컬 PDF 파일을 사용합니다
```

**A. 최신 버전 선택 시**

- WebFetch로 `https://heeyounggoo.github.io/resume` 내용을 가져온다

**B. 구버전 선택 시**

아래 순서로 캐시를 확인한 후 필요할 때만 PDF를 읽는다. PDF를 매번 읽으면 느리고 불필요하기 때문이다.

1. `.claude/skills/interview-prep/resume/resume_cache.md` 존재 여부 확인
2. 캐시가 있으면 캐시 내용을 그대로 사용한다 → 1-2로 진행
3. 캐시가 없으면: PDF를 읽고 `.claude/skills/interview-prep/resume/resume_cache.md`에 저장한다

`.claude/skills/interview-prep/resume/resume_cache.md` 형식:

```
{PDF에서 추출한 이력서 내용 전문}
```

파일이 없으면 안내: ".claude/skills/interview-prep/resume/ 폴더에 PDF 파일을 추가해주세요"

이력서를 성공적으로 로드한 후 주요 영역(프로젝트, 기술 스택, 경험 등)을 파악해 두고 1-2로 진행한다.

### 1-2. 지원 정보 수집

`.claude/skills/interview-prep/resume/jd.md` 파일 존재 여부를 확인한다.

- **파일이 있는 경우**: 읽어서 지원 회사명, JD 주요 요구사항을 파악하고 세션에 기억한다. 사용자에게 별도로 묻지 않는다.
- **파일이 없는 경우**: 묻지 않고 패스한다.

### 1-3. 누적 패턴 로드

`.claude/skills/interview-prep/memory/pattern_history.md` 파일 존재 여부를 확인한다.

- **파일이 있는 경우**: 읽어서 이전 세션들의 패턴(강점, 부족한 부분, 답변 구조/구체성 경향)을 파악하고 세션에 기억한다. 이후 답변 분석 시 이 패턴을 참조한다.
- **파일이 없는 경우**: 패스한다.

이후 Step 2로 진행한다.

## Step 2: 예상 질문 & 답변 수집

### 2-1. 질문과 답변이 있는 경우

예상 질문과 답변을 입력받고 Step 3으로 진행한다.

### 2-2. 질문이 없는 경우

1. 이력서에서 주요 영역을 추출한다 (예: 프로젝트 경험, 기술 스택, 트러블슈팅, 협업/리더십, 성장 과정 등)
2. AskUserQuestion 도구로 영역 선택지를 제시한다 (이력서에서 추출한 실제 영역 기반, 최대 4개):
   ```
   질문: "어떤 영역을 준비할까요?"
   옵션: [이력서에서 추출한 주요 영역들]
   ```
3. 선택한 영역에서 예상 질문 1가지를 제시한다
4. 그 질문에 대한 답변을 입력받는다

---

## Step 3: 질문 검증 & 답변 분석

### 3-1. 질문 검증

아래 기준으로 이 질문이 면접 대비에 의미 있는지 판단한다:

| 기준                   | 설명                                                        |
| ---------------------- | ----------------------------------------------------------- |
| **이력서 연관성**      | 이력서의 특정 경험/프로젝트/기술과 연결해 답변할 수 있는가? |
| **경력 수준 적합성**   | 7년차 FE 개발자에게 적합한 깊이와 수준인가?                 |
| **차별화 가능성**      | 이력서의 강점이나 특이한 경험을 어필할 수 있는 질문인가?    |
| **JD 매칭도**          | JD가 있을 때, 포지션 요구사항과 연관된 질문인가?            |
| **구체적 답변 가능성** | 추상적 답변이 아닌 실제 사례로 답변 가능한가?               |

**검증 결과 처리:**

- **의미 없는 질문**: 이유를 간략히 설명하고, 더 나은 질문 1-2개를 제안한 뒤 AskUserQuestion 도구로 확인한다:
  ```
  질문: "어떻게 진행할까요?"
  옵션:
    - 원래 질문으로 계속
    - 제안 질문 1로 변경: {제안 질문 1}
    - 제안 질문 2로 변경: {제안 질문 2}
  ```
- **의미 있는 질문**: 3-2로 진행한다.

### 3-2. 답변 분석 & 개선

답변을 아래 기준으로 검증하고 개선한다:

- 이력서 기반으로 논리적인가?
- 기술적으로 정확한가?
- 질문의 핵심에서 벗어나지 않는가?
- 7년차 수준에 맞는 깊이인가?

**출력 형식 (이 형식을 그대로 사용할 것):**

```
예상 질문: {예상 질문}
기존 답변: {나의 답변}
최종 답변: {수정된 답변}
주요 확인 필요:
    {좋았던 부분} - 있다면 작성
    {부족한 부분} - 있다면 작성
    {수정한 부분} - 있다면 작성
예상 꼬리 질문: {예상 꼬리 질문 2-3개}
```

**누적 패턴 참조 (pattern_history.md가 있는 경우):**

이전 세션에서 반복된 문제가 이번 답변에서도 나타나면, 분석 마지막에 아래 형식으로 명시한다:

```
⚠️ 반복 패턴 감지: {이전 세션과 동일하게 나타난 문제점} — {구체적으로 어떤 부분에서 반복됐는지}
```

분석 완료 후 Step 4로 진행한다.

## Step 4: 다음 단계 선택

분석 완료 후 AskUserQuestion 도구로 선택지를 제시한다:

```
질문: "다음은 어떻게 할까요?"
옵션:
  - 이어서 대화: 현재 Q&A에 대해 더 논의합니다
  - 꼬리 질문 답변: 예상 꼬리 질문을 선택해 답변 준비합니다
  - 새로운 예상 질문: 새 질문으로 Step 2부터 재시작합니다
```

### 4-1. 저장 시점

이 질문에 대한 논의가 완전히 끝난 시점에 `.claude/skills/interview-prep/tmp/qa_log.md`에 저장한다. 이유는 이어서 대화하는 동안 답변이 더 다듬어질 수 있기 때문이다.

- **꼬리 질문 답변 또는 새로운 예상 질문 선택 시**: 이전 Q&A가 완료된 것으로 판단하고 저장 후 진행
- **이어서 대화 선택 후**: 사용자가 "다른 질문", "다른 내용 연습", "대화 종료", "새로운 질문" 등 이 질문을 끝내겠다는 의사를 표현할 때 저장 후 Step 4 선택지를 다시 제시한다

### 4-2. 꼬리 질문 선택 시

**① 꼬리 질문 선택**: AskUserQuestion 도구로 꼬리 질문 선택지를 제시한다 (생성된 꼬리 질문 목록 기반):

```
질문: "어떤 꼬리 질문을 연습할까요?"
옵션: [생성된 꼬리 질문 2-3개]
```

**② 답변 방식 선택**: 질문을 선택한 후 AskUserQuestion 도구로 진행 방식을 묻는다:

```
질문: "어떻게 진행할까요?"
옵션:
  - 내 답변 제출: 직접 답변을 작성해서 분석받습니다
  - 예상 답변 먼저: AI가 먼저 예상 답변을 제시합니다
```

- **내 답변 제출 선택 시**: 답변을 입력받은 후 Step 3-2 답변 분석으로 진행한다 (질문 검증 단계 생략)
- **예상 답변 먼저 선택 시**: 이력서 기반으로 모범 답변 예시를 먼저 제시한 뒤, 사용자 답변을 입력받아 Step 3-2 답변 분석으로 진행한다

## Step 5: 세션 메모리

### 5-1. 저장 위치

```
.claude/skills/interview-prep/
├── tmp/
│   ├── session.md          # 세션 메타 정보 (세션 시작 시 1회 생성, 종료 시 삭제 가능)
│   └── qa_log.md           # Q&A 기록 (Q&A 완료마다 누적, 종료 시 삭제 가능)
└── memory/
    └── pattern_history.md  # 세션별 패턴 누적 기록 (삭제하지 않음)
```

### 5-2. session.md 형식

```markdown
# 면접 대비 세션

- 날짜: {오늘 날짜}
- 이력서: {최신/구버전}
- 지원 회사: {jd.md에서 읽은 회사명 또는 미확인}
- JD 키워드: {jd.md에서 읽은 주요 키워드 또는 없음}
```

### 5-3. qa_log.md 누적 형식

Q&A 완료마다 아래 블록을 추가한다:

```markdown
## Q{번호}: {질문 요약}

- 원본 답변: {기존 답변 요약}
- 최종 답변: {개선된 답변}
- 꼬리 질문: {생성된 꼬리 질문들}

---
```

### 5-4. 세션 종료

사용자가 종료를 요청하면 아래 순서로 진행한다.

**① 패턴 분석 (qa_log.md에 Q&A가 2개 이상인 경우에만 수행)**

세션 중 쌓인 Q&A 전체를 바탕으로 패턴을 분석하고 `qa_log.md` 맨 끝에 추가한다:

```markdown
---

## 세션 패턴 분석

### 강점

- {반복적으로 잘 드러난 어필 포인트들}

### 부족한 부분

- {반복적으로 수정이 필요했던 요소들}

### 개선 포인트

- {다음 연습에서 집중하면 좋을 구체적 액션들}

### 답변 구조 패턴

- {STAR(상황→과제→행동→결과) 구조 활용 여부, 두서없이 시작하는 경향 등}

### 구체성 패턴

- {수치/데이터/사례 활용 수준 vs 추상적 표현 빈도}
```

**② 누적 패턴 저장**: 분석한 패턴을 `.claude/skills/interview-prep/memory/pattern_history.md`에 누적 저장한다. 이 파일은 세션이 끝나도 삭제하지 않고 다음 세션에서 참조한다.

형식:

```markdown
## {날짜} 세션

### 강점

- {항목들}

### 부족한 부분

- {항목들}

### 답변 구조 패턴

- {항목들}

### 구체성 패턴

- {항목들}

---
```

파일이 이미 있으면 기존 내용 아래에 추가한다.

**③ 파일 삭제 확인**: AskUserQuestion 도구로 확인한다:

```
질문: "세션 파일을 삭제할까요?"
옵션:
  - 삭제: session.md, qa_log.md를 삭제합니다 (resume_cache.md는 유지)
  - 유지: 파일을 남겨둡니다
```

동의 시 `session.md`, `qa_log.md`만 삭제한다. `resume_cache.md`는 다음 세션 재사용을 위해 유지한다.

## 주의사항

- 이력서에 있는 실제 경험만 기반으로 답변을 보강한다. 없는 경험을 추가하지 않는다.
- 개선된 답변은 사용자의 말투와 스타일을 최대한 유지한다.
- 기술적 검증 시 확신이 없는 내용은 "확인 필요"로 표시한다.
- 답변이 지나치게 길어지지 않도록 실제 면접 환경(1-3분 답변)을 기준으로 한다.
